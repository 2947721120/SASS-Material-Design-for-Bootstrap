The App Bar is an alternative to the Bootstrap Navbar. It is intended to be more flexible and suit a variety of requirements.

Basics

Use nav with an .app-bar class.

Adding .app-bar-translucent will give the App Bar a 0.95 opacity.

By Default, the App Bar is a fixed height of 65px. You can optionally increase the size by adding an .app-bar-extended class to the container which increases the height to 255px.

<nav class="app-bar"></nav>

<nav class="app-bar app-bar-translucent"></nav>

<nav class="app-bar app-bar-translucent app-bar-extended"></nav>

Background Images

Use data-bg-image to specify an image URL to use as a background.

Use data-bg-position to set the background position.

See the App Bar on this page for an example. 

<nav class="app-bar app-bar-translucent app-bar-extended"
	data-bg-image="/assets/images/reading.jpg"
	data-bg-position="center center">
</nav>

Tabs

An App Bar can contain tabs. When they are used, you must add the .app-bar-has-tabs to the nav. This extends the nav so that extended navs that aren't static (ie they grow smaller as the user scrolls) maintain a height in which the tabs still display.

You probably want to add a <div class="container" /> to the tabs to center them on screen.

Tabs can be coloured using the base colour scheme (i.e. you don't define the gradient, we use fixed colours).

<nav class="app-bar app-bar-has-tabs">
	<ul class="nav nav-tabs nav-tabs-orange">
		<!-- tab li's -->
	</ul>
</nav>
<nav class="app-bar app-bar-extended app-bar-has-tabs">
	<ul class="nav nav-tabs nav-tabs-orange">
		<!-- tab li's -->
	</ul>
</nav>

Behaviour

The App Bar will be fixed to the top of the page (taking account of whether or not a Status Bar is present).

If the App Bar is extended, it will, by default, shrink when the user moves the page (until it reaches the default height). If you would like it to remain fixed, add .app-bar-static to the container class.

If you have tabs in the App Bar (see below), it shrinks to 115px, not the default (65px). 

<!-- App Bar doesn't change height when the page scrolls -->
<nav class="app-bar app-bar-extended app-bar-static">

</nav>

Floating Action Button

By adding a Floating Action Button you make a very bold action clear to the user. When you use it as part of the App Bar, it is fixed in position on the right hand side.

<nav class="app-bar app-bar-has-tabs">
	<a class="btn btn-fab btn-md bg-orange">
		<i class="icon mdi-content-add"></i>
	</a>
</nav>

Action Buttons

The Action Buttons are fixed to the top right of the app bar and offer a way for the user to access common functionality.

They should be wrapped in a div with a .action-icons class which help position them correctly.

An anchor item within that which has the .search-icon class has special functionality. When pressed, the nav toggle, title and action buttons are removed and replaced with a search interface. In addition, an overlay becomes active to focus the users gone.

When adding a search icon, you also need to include the wrapper for the search form and the .app-bar-overlay.

<nav class="app-bar">

	<div class="action-icons">
		<a href="#" class="search-icon">
			<i class="icon mdi-action-search icon-lg"></i>
		</a>
		<a href="#">
			<i class="icon mdi-action-favorite icon-lg"></i>
		</a>
		<a href="#">
			<i class="icon mdi-action-view-module icon-lg"></i>
		</a>
	</div>

	<div class="app-bar-overlay"></div>

	<div class="search-form">
		<a href="#" class="back-icon">
			<i class="icon mdi-navigation-arrow-back"></i>
		</a>
		<input type="text" class="form-control" placeholder="Search">
	</div>

</nav>
